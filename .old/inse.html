Introduction to Software Engineering

Week 1

The first lecture in INSE largely covered the way in which the assessments are conducted. The unit covers topics surrounding: teamwork, programming and documentation, in environments that seek to develop complex software systems. This will require confident use of Git/GitHub as coursework contributions are evaluated by Git commits.

This week featured no practical tutorial.

Week 2

In the lecture this week we looked at Software Process Models. We looked at the Waterfall, Incremental Development, and Reuse Oriented Software Engineering models and their advantages and disadvantages in satisfying the fundamental activies of:

  • Specification
  • Design
  • Implementation
  • Validation
  • Evolution
and how the models can accomodate documentation and configuration management.

In the tutorial this week we met our project groups and were given our project brief. Our project consists of developing a Shopping Price Comparison App.

Week 3

In this week's lecture, we looked extensively at Agile Software Development Models. A useful reference might be the Agile Manifesto, which lists the ways in which a development model can be 'agile'.

The Agile development philosophy can be separated into separate sub-methodologies. These can include: Extreme Programming, Scrum, and Feature-Driven Development. Each provide a slightly different perspective on how agility can be achieved.

Week 4

In the lecture this week, we looked at the case study of the failure of the London Ambulance Services systems update and the context for it in the process of software engineering. An important question to consider is: Where does reponsibility sit in largescale projects where no one if made officially accountable.

In the tutorial session, we simulated the process of software development within a corporation located across many officies in many countries.
